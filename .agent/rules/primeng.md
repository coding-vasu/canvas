# AI Assistant Rules for PrimeNG Development

## PrimeNG Documentation Standards

When answering questions about PrimeNG components, APIs, or usage:

1. **Always search official PrimeNG documentation first**
   - Primary source: https://primeng.org/llms/llms-full.txt (complete documentation index)
   - Secondary source: https://primeng.org/llms/llms.txt (key pages)
   - Individual pages: Add `.md` to any page URL for markdown format
   - Example: `https://primeng.org/button` becomes `https://primeng.org/button.md`

2. **Verify component APIs and properties**
   - Check the official documentation before suggesting component properties
   - Ensure import paths are correct
   - Verify that features exist in the version being used

3. **Provide accurate examples**
   - Base examples on official PrimeNG documentation
   - Include necessary imports and module declarations
   - Follow PrimeNG's recommended patterns and best practices

## Project Context

- **Framework**: Angular
- **UI Library**: PrimeNG
- **Always check**: Component properties, events, and styling options from official docs

## When to Search

- User asks about a specific PrimeNG component
- Uncertain about component API or properties
- Need to verify PrimeNG best practices
- Troubleshooting PrimeNG-specific issues

## Response Format

- Provide code examples when relevant
- Link to official documentation when helpful
- Mention version compatibility if relevant
